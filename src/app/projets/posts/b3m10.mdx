## Objectifs de cette mission

- Comprendre l'isolation : Différencier une Machine Virtuelle (OS complet) d'un Conteneur (processus isolé).
- Industrialiser le déploiement : Utiliser docker compose pour orchestrer l'application et la base de données.
- Appliquer le "Hardening" (Durcissement) :
        Configurer un utilisateur non-privilégié dans le Dockerfile.
        Limiter les ressources (CPU/RAM) pour éviter les attaques par déni de service (DoS).
        Isoler les réseaux (Backend vs Frontend).
- Auditer les vulnérabilités : Apprendre à scanner une image Docker pour détecter des failles de sécurité (CVE) avant le déploiement.
